Заметки к v3.2.0

- добавлен шаг `preflight` для ранней проверки ОС и обязательных команд
- bootstrap теперь поддерживает `--dry-run`, `--step`, `--from-step` и `--list-steps`
- команды изменения системы переведены на общий `run_cmd`, чтобы `dry-run` был реальным режимом просмотра
- отчёт расширен до CSV и JSON с данными о FQDN, роли, домене, CIFS, таймзоне и серийном номере
- в security-часть добавлен базовый hardening для `firewalld` и `sshd`
- добавлены новые конфигурационные параметры для firewall и SSH
- bootstrap стал мягче работать в окружениях без `flock`

Развитие toolkit v3 завершено.
Заметки к v3.1.0

- добавлен внешний шаблон конфигурации через `config.sh.example`
- добавлен `README.md` с инструкцией по запуску и текущими ограничениями
- большинство скриптов переведено на `set -euo pipefail`
- ad-hoc обработка lock-файла в bootstrap заменена на `flock`
- убран автоматический `git pull` во время self-update, чтобы избежать смешанных версий кода
- обновление proxy-конфигурации сделано идемпотентным
- добавлена проверка на уже выполненный ввод в домен
- опасная очистка repo-конфигурации заменена явной безопасной заглушкой
- добавлен механизм локальных hooks через `custom/*.local.sh`
- добавлен `scripts/validate.sh` для синтаксической проверки и линтинга
- добавлен GitHub Actions workflow для проверки shell-скриптов
